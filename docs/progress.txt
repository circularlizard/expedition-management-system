Tasks Completed (Spec 3: Signup Data Model & Fluent Forms Sync):
1. Migrated `ems_signups` table via `EMS\Core\Table_Installer` with full schema (scout_id, unit_id, dofe_level, payment_status, etc.).
2. Implemented `EMS\Integrations\Fluent_Forms_Sync` with seven FF hooks: rendering_field_data_select, validate_input_item_select, validation_errors, submission_inserted, after_payment_status_change, before_form_render, and rendering_field_data_input_email (×3 for email field pre-population).
3. Implemented `EMS\Data\Signup_Repository` (create, get, get_by_submission_id, update_payment_status, get_all).
4. Email notifications delegated entirely to Fluent Forms' built-in notification system (no wp_mail calls from EMS). EMS pre-populates three hidden email fields on render and updates them via JS when the child selector changes:
   - signup_parent_email   — WP user account email (always available)
   - signup_explorer_email — ems_osm_explorers.email if child is synced, else empty
   - signup_leader_email   — ems_units.leader_email for resolved unit, else empty
   Admins configure FF notification rules to use these fields as smart tag recipients.
5. Client-side sync JS: `window.emsFormMappings` includes unitCode, unitId, explorerEmail, leaderEmail per child. Uses jQuery(el).data('choicesjs') with 3 s polling retry for fluentform_init timing race.
6. resolve_unit_for_child() extended to return leader_email alongside short_code and unit_id.
7. Bug fix — Choices.js sync: `el.choicesInstance` is always undefined in Fluent Forms (instance stored under jQuery `.data('choicesjs')`). Fixed lookup and added polling retry.
8. Bug fix — Payment status mapping: `completed` was dead code (not a real FF status). Corrected map: `paid`/`succeeded` → 'paid'; all else → 'pending'. Added idempotency guard to block downgrade of already-paid rows. Removed debug `file_put_contents` logger.
9. Updated Gherkin feature (`signup-fluentforms-sync.feature`): uses correct Stripe status `paid` (not `completed`), added two new idempotency scenarios.
10. 13 PHPUnit tests green (Fluent_Forms_SyncTest suite).
